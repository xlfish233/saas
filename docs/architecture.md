## 核心设计原则

### 1. Cloud-Native First
- 容器化工作负载
- Kubernetes 编排
- 服务网格就绪

### 2. Multi-Tenant by Design
- 每个请求携带租户上下文
- 数据库级数据隔离
- 每租户资源配额

### 3. API-First
- RESTful + GraphQL API
- OpenAPI 文档
- API 版本控制

### 4. Event-Driven
- 领域事件用于服务间通信
- 事件溯源用于审计追踪
- CQRS 读写分离

### 5. Resilience
- 熔断器
- 指数退避重试
- 优雅降级

### 6. Observability
- 结构化日志
- 分布式追踪
- 每租户指标

### 7. Zero-Trust Security
- 永不信任，始终验证
- 所有服务通信使用 mTLS
- 最小权限访问
- 基于身份的安全边界

### 8. Infrastructure as Code
- 声明式基础设施 (Terraform)
- GitOps 部署 (ArgoCD)
- 不可变基础设施
- 环境一致性 (12-Factor App)

## 技术栈

| 层级 | 技术 | 版本 |
|-----|-----|-----|
| Web Framework | Axum | 0.8 |
| Async Runtime | Tokio | 1.x |
| Database | SQLx + Aurora PostgreSQL | 0.8 / 15 |
| Cache | Redis | 7.x |
| Message Queue | NATS JetStream | 2.x |
| Object Storage | S3 / MinIO | - |
| Container Runtime | Docker / containerd | - |
| Orchestration | Kubernetes (EKS) | 1.28+ |
| Service Mesh | Istio (可选) | 1.20+ |
| Monitoring | Prometheus + Grafana | - |
| Tracing | OpenTelemetry + Jaeger | - |

```
                           [路由到对应隔离层]
```
